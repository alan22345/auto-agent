# Search Tab — Design Spec

**Date:** 2026-04-30
**Status:** Draft, pending implementation plan

## Goal

Add a **Search** tab to `web-next` that hosts a multi-turn chat with a research
agent. Each turn, the agent can search the web (Brave), fetch a URL, recall
facts from the shared team-memory graph, and save new facts when warranted.
Reasoning, sources, and the final answer stream into the UI as they are
produced.

This replicates the user-facing capability of `ergodic-ui/app/search` but is
implemented as an agent flow inside the auto-agent runtime — not as a
hand-rolled brave→summarize→synthesize pipeline.

## Non-goals (v1)

- Thumbnails, favicons, page age, or per-source relevance ratings.
- Shareable / slug-based search URLs (sessions are accessed via the session
  list).
- Automatic post-turn fact extraction. The agent decides when to call
  `remember_memory`; no background extractor runs.
- Multi-user session sharing.

## Architecture

```
web-next/app/(app)/search/page.tsx          Tab entry — session list + chat pane
web-next/components/search/                 Message list, source list, input box
web-next/hooks/useSearchStream.ts           Opens NDJSON stream, parses events

  │  POST /search/sessions/{id}/messages   (NDJSON stream)
  ▼

orchestrator/search.py                      New router — sessions CRUD + streaming
shared/models.py (extended)                 SearchSession, SearchMessage
migrations/versions/<new>.py                Alembic migration

agent/search_loop.py                        Wrapper around agent/loop.py:
                                            • search-specific system prompt
                                            • tool subset
                                            • emits NDJSON events to caller

agent/tools/web_search.py                   Brave API → list[{url, title, description}]
agent/tools/fetch_url.py                    requests + BS4 → cleaned text (truncated)
agent/tools/recall_memory.py                Wraps GraphEngine.recall
agent/tools/remember_memory.py              Wraps GraphEngine.remember
```

The Search tab reuses the existing agent loop, provider, and context pipeline.
What is new is a curated tool subset, a chat-shaped persistence layer, and the
streaming HTTP surface.

## Data model

```python
# shared/models.py

class SearchSession(Base):
    id: int (pk)
    user_id: int (fk → User)
    title: str            # "New search" on create; replaced after the first assistant turn
                          # by a one-line title generated by a single short LLM call (non-blocking)
    created_at: datetime
    updated_at: datetime

class SearchMessage(Base):
    id: int (pk)
    session_id: int (fk → SearchSession, indexed, ordered)
    role: Literal["user", "assistant"]
    content: str          # for user: raw text; for assistant: final markdown answer
    tool_events: JSON     # list of {tool, args, result_summary, ts} captured during turn
    created_at: datetime
```

Sessions belong to the authenticated user. The agent loop receives the full
prior message history of the session as context on every turn (multi-turn
chat).

## HTTP surface

All routes are auth-gated using the existing session cookie auth.

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/search/sessions` | Create a new session. Returns `{id, title}`. |
| `GET`  | `/search/sessions` | List the user's sessions, newest first. |
| `GET`  | `/search/sessions/{id}` | Return a session with its full message history. |
| `DELETE` | `/search/sessions/{id}` | Delete a session and its messages. |
| `POST` | `/search/sessions/{id}/messages` | Send a user message; stream the assistant turn back as NDJSON. |

The streaming endpoint persists the user message before the stream begins, and
persists the assistant message (with `tool_events`) when the stream completes.
On client disconnect mid-stream, the partial assistant message is still
persisted with a `truncated` marker in `tool_events`.

## Streamed event schema

The streaming endpoint emits NDJSON, one JSON object per line, with these
event types:

| `type` | Payload | Emitted when |
|---|---|---|
| `tool_call_start` | `{id, tool, args}` | Agent begins a tool call. |
| `source` | `{url, title, summary, query}` | One row per Brave result, emitted from inside `web_search`. |
| `memory_hit` | `{entity, facts}` | One row per `recall_memory` match. |
| `text` | `{delta}` | Assistant token delta. |
| `done` | `{message_id}` | Turn complete; assistant row persisted. |
| `error` | `{message}` | Aborted. Stream closes after this event. |

The frontend uses `tool_call_start` for activity indicators ("Searching the
web…", "Reading example.com…") and uses `source` / `memory_hit` to populate the
source / memory blocks under the assistant message as the turn progresses.

## Agent tools

Each tool follows the existing `agent/tools/base.py` `Tool` contract.

### `web_search`

- Input: `{query: str, num_results: int = 8}`
- Calls Brave Search API with the configured API key.
- Returns a list of `{url, title, description}` (Brave's snippet, no scrape).
- Side effect: emits one `source` event per result via the loop's event sink,
  so the UI can render source cards before the agent finishes reading them.

### `fetch_url`

- Input: `{url: str}`
- HTTP GET with a real User-Agent, BS4 to strip script/style, html2text to
  render markdown, truncate to ~32k characters.
- Returns `{url, title, content}`.
- Used when Brave's description is insufficient and the agent needs the page
  body.

### `recall_memory`

- Input: `{query: str}`
- Calls `team_memory.graph.GraphEngine.recall` against the shared atlas DB.
- Returns matched entities and facts, or `{ambiguous: true, candidates: [...]}`.
- Side effect: emits a `memory_hit` event per match.

### `remember_memory`

- Input: `{entity_name: str, entity_type: str, fact: str, kind: str}`
- Calls `GraphEngine.remember`.
- **Tool description constrains usage tightly:**
  > Use ONLY when (a) the user has explicitly asked you to remember
  > something, or (b) the user has stated a durable preference or fact about
  > themselves or the project that will be useful in future conversations.
  > Do NOT use this tool to save research findings, web search summaries, or
  > anything you learned from `web_search` / `fetch_url`. Those belong in the
  > current search session, not in team-memory.
- Returns `{ok: true, fact_id: ...}`.

## Team-memory integration

1. **Pre-turn recall (automatic).** Before the agent runs, call
   `agent.context.memory.query_relevant_memory(user_message)` and inject the
   result into the system prompt under a "Shared Team Memory" header.
2. **Agent-driven recall.** Via the `recall_memory` tool above. Emits
   `memory_hit` events to the UI.
3. **Agent-driven remember.** Via the `remember_memory` tool above, with the
   tight description that excludes web findings.
4. **No post-turn extraction.** The existing `agent/memory_extractor.py` is
   not run after search turns in v1. (Trivial to enable later behind a
   feature flag if we want it.)

## UI shape

### Sidebar

`components/sidebar/sidebar.tsx`: append `{ href: '/search', label: 'Search' }`
to the `tabs` array. No icon (matches existing tabs).

### Search page (`app/(app)/search/page.tsx`)

Two-column layout:

- **Left rail (~260px):** session list, ordered by `updated_at` desc. "New search" button at
  the top creates a fresh session and navigates to it. Each row shows
  `title` and a relative timestamp; right-click / hover reveals delete.
- **Right pane:** the chat for the active session. Empty state shows a brief
  prompt ("Ask anything — I'll search the web and our team memory.") plus
  the input box centered.

### Chat pane

- Scrollable message list, oldest at the top, auto-scrolls to bottom on new
  events.
- **User messages:** plain text bubble.
- **Assistant messages:**
  - Activity strip at top showing the active tool call ("Searching: …",
    "Reading example.com", "Recalling team memory: …") while streaming. Strip
    fades out when `done` arrives.
  - Markdown-rendered answer body. We don't have `react-markdown` in
    `web-next` yet — add it as a dependency.
  - Collapsible **Sources (N)** group below the answer. Each row: `[N]
    title` (clickable, opens in new tab), then a one-line summary on the
    line below. URL hostname shown dimmed at the right.
  - Collapsible **From team memory (N)** group, only shown when at least one
    `memory_hit` event fired. Each row: `[entity_type] entity_name` and the
    fact text below.
- **Input box:** at the bottom, sticky. Enter submits, Shift+Enter inserts a
  newline. Disabled while a turn is streaming. A small "Stop" button next to
  it cancels the stream (closes the fetch + tells the backend to abort).

### Streaming hook

`hooks/useSearchStream.ts` opens a fetch to the messages endpoint, reads the
body as a stream, splits on `\n`, parses each line as JSON, and dispatches
into a reducer. State shape:

```ts
type StreamState = {
  status: 'idle' | 'streaming' | 'error' | 'done';
  activeTool: { tool: string; args: any } | null;
  sources: SourceEvent[];
  memoryHits: MemoryHitEvent[];
  answer: string;       // accumulated text deltas
  error: string | null;
};
```

## Configuration

New entry in `shared/config.py`:

```python
brave_api_key: str = ""   # Required; /search returns 503 with a clear error if unset
```

No Firecrawl, no other new env vars.

## Error handling

- Missing `BRAVE_API_KEY`: `/search/sessions/{id}/messages` returns `503` with
  body `{"error": "Search is not configured. Set BRAVE_API_KEY."}` before
  starting the stream.
- Brave API error: `web_search` tool returns an error result; agent sees it
  and can decide to retry with a different query or fall back to memory.
- `fetch_url` failure (timeout, 4xx/5xx, unparseable): tool returns an error
  result; agent decides what to do.
- Agent loop exception: stream emits an `error` event and closes; assistant
  message is persisted with whatever partial content was streamed.
- Team-memory unavailable (`team_memory_session is None`): `recall_memory` and
  `remember_memory` return a clear error; the agent continues without memory.

## Testing

- **`tests/test_web_search_tool.py`:** mocked Brave HTTP; assert events
  emitted, result shape correct, error path returns useful message.
- **`tests/test_fetch_url_tool.py`:** mocked HTTP; assert truncation, title
  extraction, error handling.
- **`tests/test_recall_remember_tools.py`:** against an in-memory team-memory
  fixture; assert `recall_memory` emits `memory_hit` events and
  `remember_memory` writes a fact.
- **`tests/test_search_endpoint.py`:** end-to-end with a mocked agent loop
  that emits canned events; assert NDJSON line-by-line shape, persistence of
  user + assistant rows, partial-persist on disconnect.
- **`web-next/__tests__/useSearchStream.test.ts`:** feed canned NDJSON into
  the hook; assert reducer state at each step.
- **Frontend e2e (Playwright, optional):** create a session, send a message
  with the agent loop stubbed to a canned stream, assert sources and answer
  render.

## Open questions / future work

- Background memory extraction over the assistant reply, gated by a per-user
  setting.
- Source citations in the answer: have the agent inline `[1]`-style
  references that hyperlink to source rows. Possible v1.5 enhancement.
- Search history search ("find that thing I asked about last week"): probably
  belongs in team-memory rather than as a separate index.
