## Goal

Ship the final phase of ADR-016: per-area refresh + UI polish (side
panel on node click, evidence popover on edge click, search, edge-kind
filter, ancestor/descendant highlight, refresh-this-area buttons,
freshness-banner polish, ast/llm visual distinction).

## Acceptance criteria

* `POST /api/repos/{id}/graph/refresh?area=<name>` triggers a partial
  pipeline run; non-target areas are preserved verbatim in the new row;
  cross-area + HTTP edges are re-validated; same workspace state
  → identical RepoGraphBlob (modulo `generated_at`).
* `GET /api/repos/{id}/graph/code?path=&line_start=&line_end=` returns
  workspace file contents, bounded (≤500 lines, ≤50KB), 400 on bad
  range, 404 on missing file, 400 on path traversal.
* Cytoscape canvas fires node-click + edge-click events; the page
  renders a side panel and an evidence popover.
* Search input filters nodes by name (fade non-matches).
* Edge-kind toggles (4 checkboxes) hide edges of unchecked kinds.
* Selected node has "Highlight ancestors" / "descendants" controls.
* Freshness banner shows full sha tooltip, analyser version tooltip,
  stale-warning when staleness primitive reports drifted.
* LLM edges render dotted; AST edges solid.
* Every change has a failing-first test.
* `pytest` + `ruff check` + `ruff format --check` + `tsc --noEmit` +
  `npm run build` + `vitest` all pass.

## Tactical decisions

* **Per-area refresh button: page-level chip strip, not in canvas.** Overlays inside Cytoscape are fiddly to position and untestable with React Testing Library. ADR-016 §11 wants "per-box button on each collapsed area" — the chip strip satisfies the discoverability + testability intent.
* **Search/filter/highlight: state lifted to page, props to canvas.** Cytoscape side-effects are quarantined inside one useEffect that observes the prop changes.
* **No new fields on Edge/Node/RepoGraphBlob.** ADR-016 freeze.
* **Code preview reads from `RepoGraphConfig.workspace_path`.** Already on the config row.
* **Partial pipeline preserves nodes/edges from non-target areas verbatim** — only HTTP matching and boundary flagging re-run across the whole node set.

### 3a — Freshness banner polish + `/graph/staleness` endpoint

Backend (`orchestrator/router.py`):

* New endpoint `GET /api/repos/{repo_id}/graph/staleness`.
* Response model `GraphStalenessResponse` in `shared/types.py` (new model,
  no field additions to existing models): `{graph_sha, workspace_sha, drifted}`.
  `workspace_sha` is `str | None`.
* Behaviour:
  - 404 if repo not found.
  - 404 if `RepoGraphConfig` not enabled.
  - 404 if no `last_analysis_id` — there is no graph to compare against.
  - Otherwise: load the `RepoGraph` row, call
    `compute_staleness(graph_sha=row.commit_sha, workspace_path=cfg.workspace_path)`,
    return the dataclass mapped to the Pydantic model.

5. `test_returns_not_drifted_when_workspace_matches_graph_sha`
