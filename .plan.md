# ADR-016 Phase 3 — LLM gap-fill + citation validation + bounded escape

## Goal

Resolve dynamic-dispatch sites that Phase 2's tree-sitter parser counts but
cannot prove. For each detected `UnresolvedSite` the pipeline runs a one-shot
LLM call, validates every emitted edge's citation against the actual file, and
falls back to a bounded mini agent-loop (3 tools, max 5 iterations, max 60s,
32k input + 4k output token budget) only when the one-shot path fails. Edges
that survive carry `source_kind="llm"` and ride the existing `RepoGraphBlob`
schema unchanged.

## Acceptance criteria

- `agent/graph_analyzer/parsers/__init__.py::Parser` now returns a
  `ParseResult` with an `unresolved_sites: list[UnresolvedSite]` field
  alongside `nodes` and `edges`. `unresolved_dynamic_sites` integer count
  derives from `len(unresolved_sites)` — no separate counter.
- Python parser populates `unresolved_sites` for every previously-counted
  dynamic site: registry-dict dispatch (`HANDLERS[name](payload)`),
  attribute calls on imported modules (`mod.something()`),
  `self.unknown_method()`, unbound identifier calls, attribute-class
  inheritance. Each site carries workspace-relative file path, 1-indexed
  line, the call snippet, the id of the containing function/method, ~30
  lines of surrounding source, and a `pattern_hint` string
  (`"registry" | "getattr" | "decorator_routed" | "dict_call" | "unknown"`).
- `agent/graph_analyzer/gap_fill.py::gap_fill_site` runs ONE
  `complete_json` call per site, returns `list[Edge]` (possibly empty)
  with `source_kind="llm"`, `kind="calls"`, and an `EdgeEvidence` from the
  LLM's citation.
- `agent/graph_analyzer/validator.py::validate_citation` opens the cited
  file, reads `evidence.line` ±2 lines, returns True iff
  `evidence.snippet.strip()` is a substring of any of those lines.
- `agent/graph_analyzer/validator.py::validate_target` returns True iff
  `edge.target` matches a node id in the supplied list.
- `agent/graph_analyzer/agent_escape.py::agent_escape` exposes 3 tools to
  the LLM (`read_file`, `grep`, `return_findings`) with hard caps (max 5
  iterations, max 60s wall-clock, max 32k input + 4k output tokens across
  the loop). Each emitted edge is validated identically.
- `pipeline.run_pipeline` accepts an optional `provider` argument
  (default `None` → skip gap-fill stage entirely so deterministic tests
  still pass). When `provider` is provided, the pipeline runs gap-fill per
  area between parser pass and assembly, drops edges that fail citation /
  target validation, and stamps the surviving LLM edges into the blob.
- `RepoGraphBlob` schema unchanged. No migration.
- 100% of new code paths covered by tests. Mocked provider only — no real
  network calls.

## Steps

1. **Types** — write `agent/graph_analyzer/types.py` with `UnresolvedSite`.
2. **Parser refactor** — update `Parser` ABC + `ParseResult` so that
   `unresolved_dynamic_sites: int` is gone and `unresolved_sites:
   list[UnresolvedSite]` takes its place. Update Python parser to populate
   sites with line + snippet + containing-node id + 30-line surrounding
   source + pattern hint. Existing parser tests assert `unresolved_sites`
   length instead of int counter.
3. **Validator** — `validate_citation` (fuzzy ±2) + `validate_target`,
   with tests first.
4. **Gap-fill** — `gap_fill_site` builds prompt with surrounding code +
   candidate node ids; parses JSON; constructs `Edge`s with
   `source_kind="llm"`, `kind="calls"`, source = `site.containing_node_id`.
   Tests with mocked provider.
5. **Agent escape** — focused mini-loop. Use `provider.complete` with
   `tools=[...]` definitions; route tool calls through three handlers;
   accumulate token usage and elapsed time; terminate on `return_findings`,
   on max iterations, on wall-clock, on token budget. Tests with mocked
   provider including a slow / always-loops mock.
6. **Pipeline integration** — `run_pipeline(workspace, commit_sha,
   provider=None)`. When provider is set, run the gap-fill stage per area.
   Update `AreaStatus.unresolved_dynamic_sites` to count from the
   `unresolved_sites` list. Validate edges; drop failures; include
   survivors in blob.edges. Existing tests pass unchanged because the
   default `provider=None` skips Phase 3.
7. **Phase-3 end-to-end test** — set up a tmp workspace with the registry
   fixture, run the pipeline with a mocked provider, assert the resulting
   blob contains an LLM edge with the expected source/target/evidence.
8. **Verify** — `pytest tests/`, `ruff check .`, `ruff format --check .`,
   `cd web-next && npm run build`, `tsc --noEmit`, `npx vitest run`.
9. **Commit** — focused commits with messages referencing
   `ADR-016 Phase 3 §<component>`.

## Tactical decisions

- **`complete_json` does not currently accept a `schema_hint` kwarg.**
  The signature in `agent/llm/structured.py` is `(provider, messages, *,
  system, max_tokens, temperature, retries)`. The instruction set
  references `schema_hint` as an existing parameter; in practice the
  schema is encoded into the system prompt itself. Phase 3 follows the
  existing convention — schema shape is described inside the system
  prompt and validated by Pydantic on the way out.
- **Per-area candidate pool** — Phase 3 caps each site's prompt to ≤150
  candidate node ids. Selection priority: same file, then same area,
  then anything imported from the file (best-effort match against
  `module:<path>` ids from earlier areas). This is intentionally a
  heuristic; cross-area selectivity is Phase 5's concern.
- **Pipeline back-compat** — `run_pipeline(workspace, commit_sha)` keeps
  working. Adding `provider=None` as a third kwarg means the Phase 2
  call sites (`agent/lifecycle/graph_refresh.py`) keep building blobs
  with AST-only edges until they choose to start passing a provider.
  Phase 3 ships the gap-fill capability; whether production wires it up
  is a separate decision tracked under the same ADR.
- **Failure isolation** — gap-fill exceptions for a single site never
  fail the area. They get logged and the site contributes zero edges.
  The whole-area `try` block keeps catching parser-level failures.
- **Bounded escape token accounting** — use the `usage` field on
  `LLMResponse` returned by `provider.complete`. Track cumulative
  `input_tokens` and `output_tokens`. Terminate before issuing a call
  that would exceed the budget. A mocked provider that returns 0 tokens
  per call is fine — the wall-clock and iteration bounds still apply.
- **Wall-clock bound testing** — patch the mini-loop's `time.monotonic()`
  reference (we import the function locally and dispatch via a module-
  level `_now` indirection) so tests don't have to actually sleep.
- **`return_findings` is terminal** — its handler stops the loop. Other
  tool calls that arrive in the same model turn after a successful
  `return_findings` are ignored.
- **Tool definitions live in the escape module.** They're not registered
  with the global `ToolRegistry` because they're narrow-scope and
  workspace-pinned.
- **Edge construction in escape uses the same schema as gap_fill.** Both
  paths converge on `_emit_edges_from_payload(site, payload)` to avoid
  drift.
