# Plan ‚Äî ADR-016 Phase 2 (code-graph tree-sitter pipeline)

## Goal

Ship the deterministic tree-sitter Python pipeline for ADR-016, replacing the
Phase 1 refresh stub with a real event-driven analyser that writes a
`RepoGraph` row whose `graph_json` is a `RepoGraphBlob` (locked schema ‚Äî
additive for Phase 3+). Wire `/code-graph/{repoId}` to render the resulting
hierarchical compound graph via Cytoscape with expand/collapse and a
freshness banner.

## Scope (Phase 2 only)

- Tree-sitter Python parser: files / classes / functions + `imports` /
  `inherits` / statically-resolvable `calls`.
- No LLM gap-fill ‚Äî dynamic-dispatch sites get counted, not resolved.
- `RepoGraphBlob` Pydantic schema; `source_kind="ast"` on every edge with
  real `(file, line, snippet)` evidence.
- Typed events `REPO_GRAPH_REQUESTED|READY|FAILED` and `repo_graph_*`
  factories.
- Refresh endpoint becomes 202-Accepted; publishes request event.
- Background handler (in `agent/lifecycle/graph_refresh.py`, registered in
  `agent/main.py`) clones / fetches the workspace, runs the pipeline,
  writes the row, publishes READY/FAILED.
- Per-area failure isolation; `.auto-agent/graph.yml` overrides default
  top-level-dir areas; non-source directories always skipped.
- Parser registry pattern: `agent/graph_analyzer/parsers/python.py` plus
  abstract `Parser` base ‚Äî adding TS in Phase 4 = one file.
- Cytoscape rendering with `cytoscape-expand-collapse`; areas collapsed
  by default; failed areas styled red; freshness banner + Refresh.

## Tactical decisions

- **Handler location** ‚Äî task brief named `run.py` but pointed to the
  `repo_onboard` pattern. That pattern lives in `agent/main.py`
  (`bus.on(RepoEventType.ONBOARD, harness_onboard.handle)`). Putting the
  graph-refresh handler in the **agent** event loop honours dependency
  layering (agent must not import from orchestrator) and keeps the new
  handler symmetric with the existing repo-event handler. New module:
  `agent/lifecycle/graph_refresh.py`.
- **Lock contention** ‚Äî the task brief says return 409 on duplicate
  refresh. The 202 endpoint just publishes a request event; lock
  contention is detected inside the handler (which then publishes
  `REPO_GRAPH_FAILED` with `"analysis already running"`). For the
  endpoint itself, the only synchronously-detectable contention case is
  "another request is mid-flight on this same async worker", which the
  per-repo `request_id` cannot easily detect across processes. I treat
  the endpoint as fire-and-forget ‚Äî duplicate calls always 202; the
  handler publishes FAILED if the lock can't be acquired. Tests assert
  this behaviour.
- **Concurrency between API endpoint and handler** ‚Äî the handler is on
  the agent event loop (separate process from the orchestrator in prod);
  the endpoint just publishes. The InMemoryPublisher lets tests drive
  the handler directly without Redis.
- **Workspace clone vs reset** ‚Äî match the ADR ¬ß9 exactly:
  `git clone --depth=1 -b <branch>` on first run; `git fetch origin &&
  git reset --hard origin/<branch>` on subsequent runs.
- **Subprocess git** ‚Äî use `asyncio.create_subprocess_exec` rather than
  importing `agent/tools/git.py` (which is a workspace-sandbox wrapper
  built for the in-loop agent, not a general subprocess helper).
- **Tree-sitter version** ‚Äî `tree_sitter>=0.23` with the
  `tree_sitter_python` PyPI package (the modern grammar binding;
  doesn't need `Language.build_library`). Confirmed working in
  `/Users/alanyeginchibayev/Documents/Github/auto-agent/.venv`.
- **Calls resolution** ‚Äî Phase 2 statically resolves only when the
  callee name is bound in module scope (top-level def/class or import).
  `self.method()` calls inside a class are resolved to the same-class
  method when the class is the enclosing scope. All other call sites
  (attribute access on imported objects, decorator-based dispatch,
  `getattr`, `HANDLERS[x]`) are *detected* and counted in
  `area.unresolved_dynamic_sites` ‚Äî they do not emit edges.
- **Inherits resolution** ‚Äî `class Foo(Base)` ‚Üí edge `Foo` ‚Üí `Base` if
  `Base` is in module scope (top-level def/class/import); otherwise
  dropped. `class Foo(mod.Base)` is treated as unresolved.
- **Frontend polling** ‚Äî use TanStack Query with a 3 s refetch interval
  on the graph endpoint while the latest config has no analysis or the
  banner indicates a stale state; pause the interval otherwise.

## Files

### Types & events (foundation)
1. `shared/types.py` ‚Äî add `Node`, `EdgeEvidence`, `Edge`, `AreaStatus`,
   `RepoGraphBlob`, `RepoGraphRefreshResponse`. Re-export
   `RepoGraphData` with `commit_sha`, `generated_at`, `status`.
2. `shared/events.py` ‚Äî add `RepoEventType.GRAPH_REQUESTED`,
   `.GRAPH_READY`, `.GRAPH_FAILED`; factories
   `repo_graph_requested(repo_id, request_id)`,
   `repo_graph_ready(repo_id, repo_graph_id, commit_sha, status)`,
   `repo_graph_failed(repo_id, error)`.

### Parser & pipeline
3. `agent/graph_analyzer/__init__.py` ‚Äî package init.
4. `agent/graph_analyzer/parsers/__init__.py` ‚Äî abstract `Parser` base
   class and the `parser_for(extension)` registry function.
5. `agent/graph_analyzer/parsers/python.py` ‚Äî tree-sitter Python parser:
   `parse_file(text, file_path) -> ParseResult` returning nodes,
   edges (with evidence), and `unresolved_dynamic_sites` count.
6. `agent/graph_analyzer/pipeline.py` ‚Äî orchestrates the run: area
   discovery (graph.yml or top-level dirs minus skip-list), per-area
   parsing, per-area failure isolation, blob assembly.

### Refresh wiring
7. `orchestrator/router.py` ‚Äî rewrite `refresh_repo_graph` to publish
   `repo_graph_requested` and return 202 with `request_id`. Delete the
   501-stub language. Add `GET /api/repos/{repo_id}/graph/latest` to
   return the latest `RepoGraph` row for the page (status, blob,
   commit_sha, generated_at).
8. `shared/types.py` ‚Äî `RepoGraphRefreshResponse` ({ request_id,
   status: "accepted" }), `LatestRepoGraphData` (the GET /latest
   payload).
9. `agent/lifecycle/graph_refresh.py` ‚Äî event handler. Acquires
   workspace lock; clones or fetches; resolves commit_sha; runs
   pipeline; writes `RepoGraph` row; updates
   `RepoGraphConfig.last_analysis_id` + `analyser_version`; publishes
   READY or FAILED.
10. `agent/main.py` ‚Äî register `graph_refresh.handle` against
    `RepoEventType.GRAPH_REQUESTED`.

### Frontend (web-next)

14. `web-next/components/code-graph/graph-canvas.tsx` ‚Äî Cytoscape
