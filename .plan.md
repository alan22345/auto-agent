# Phase 6 of ADR-016 — Agent interface for the code graph

Branch: `phase-6-base-adr-016`. Worktree:
`/Users/alanyeginchibayev/Documents/Github/auto-agent/.claude/worktrees/phase-6-base/`.

## Goal

Give the auto-agent a single typed tool to query the code graph and a system-prompt
nudge so it reaches for the tool before grepping. Every tool response carries a
staleness envelope and per-result workspace-existence flags so the agent can
filter out symbols that have drifted in its task branch.

## Acceptance criteria

1. `query_repo_graph(repo_id, op, params)` agent tool exists, registered in
   `agent/tools/__init__.py`, implements all seven ops (`callers_of`,
   `callees_of`, `outgoing_edges`, `incoming_edges`, `public_surface`,
   `path_between`, `violates_boundaries`).
2. Every successful response is a JSON object with `result`, `staleness:
   {graph_sha, workspace_sha, drifted}`, and (where applicable)
   `results_with_existence[*].{value, exists_in_workspace}`.
3. Tool returns a structured error result when the repo has no analysis
   (`last_analysis_id is None`) — no exceptions, `is_error=True`.
4. New helper `agent/graph_analyzer/staleness.py::compute_staleness` reads
   workspace HEAD via `git rev-parse HEAD`; returns `drifted=True` and
   `workspace_sha=None` when the workspace dir doesn't exist.
5. System-prompt builder adds the code-graph nudge paragraph IFF the repo has
   `graph_enabled=True` (a `RepoGraphConfig` row exists) AND
   `last_analysis_id is not None`. Nudge absent in either negative case.
6. Public surface for `public_surface(area)` is read from the blob, not
   re-derived. (Schema change documented below.)
7. All unit tests green; `ruff check`, `ruff format --check`, `tsc --noEmit`,
   `npm run build` clean.

## Tactical decisions

- **Public surface = blob field.** Adding `public_symbols: list[str] =
  Field(default_factory=list)` to `RepoGraphBlob`. The pipeline already
  computes `all_public_symbols` (used by `flag_violations`) — we just plumb it
  onto the blob. Legacy `graph_json` rows lack the field; the Pydantic default
  keeps them loadable. Re-deriving via re-parsing source bytes at query time
  was rejected: parsers depend on source bytes and tree-sitter, and the
  pipeline already has the answer.
- **Workspace for staleness + existence check** = the graph-analyser workspace
  (`RepoGraphConfig.workspace_path`). The tool has no task-context awareness;
  using the graph workspace is the documented fallback from the brief
  ("If you discover that determining 'the task's workspace' requires task
  context the tool doesn't have, document the decision and use the graph's
  analyser workspace as a fallback"). Future work can thread task workspace
  through `ToolContext` if useful.
- **`exists_in_workspace` check** uses `os.path.exists(workspace/<node.file>)`
  per Phase 6 v1 minimum. A node with no `file` field (area-kind nodes) gets
  `exists_in_workspace=True` — vacuously true, nothing to miss.
- **`path_between`** returns a list of node ids when a path exists; `[]`
  otherwise. `max_depth` defaults to 5; `params={"max_depth": N}` overrides.
- **`violates_boundaries`** returns a serialised `Edge` dict (or `None` if no
  edge exists) preserving `evidence`/`source_kind`/etc.
- **Tool wraps lists of nodes/edges** in `results_with_existence`; single
  values (path list, single edge, single bool) keep `result` only.
- **No caching** (ADR-016 Phase 6 explicitly defers tool-result caching).
- **No `graph_enabled` column** — the `RepoGraphConfig` model has no such
  field; presence of the row IS the opt-in (consistent with how every other
  consumer treats it). The nudge condition is therefore "config exists AND
  `last_analysis_id is not None`".
